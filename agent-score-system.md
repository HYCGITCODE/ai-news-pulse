# Agent 评分激励机制

## 一、评分维度

| 维度 | 权重 | 满分 | 说明 |
|------|------|------|------|
| **效率分** | 25% | 100 分 | 任务完成速度与计划对比 |
| **质量分** | 30% | 100 分 | 交付物质量、错误率、返工次数 |
| **沟通分** | 15% | 100 分 | 主动通知、响应时效、协作态度 |
| **流程分** | 20% | 100 分 | 流程遵守、文档更新、汇报及时 |
| **创新分** | 10% | 100 分 | 主动优化、提出改进建议 |

**综合得分计算公式：**
```
综合得分 = 效率分×0.25 + 质量分×0.30 + 沟通分×0.15 + 流程分×0.20 + 创新分×0.10
```

## 二、评分标准

### 2.1 效率分（满分 100 分）

| 评分项 | 加分规则 | 减分规则 | 评分依据 |
|--------|----------|----------|----------|
| **基准分** | - | - | 80 分（按时完成） |
| **提前完成** | +5 分/提前 10% 时间 | - | 对比计划工时 |
| **大幅提前** | +10 分/提前 30% 时间 | - | 对比计划工时 |
| **轻微延期** | - | -5 分/延期 10% 以内 | 对比计划工时 |
| **明显延期** | - | -15 分/延期 30% 以内 | 对比计划工时 |
| **严重延期** | - | -30 分/延期 50% 以上 | 对比计划工时 |
| **紧急任务响应** | +10 分/紧急任务 1h 内响应 | - | 任务响应时间记录 |

### 2.2 质量分（满分 100 分）

| 评分项 | 加分规则 | 减分规则 | 评分依据 |
|--------|----------|----------|----------|
| **基准分** | - | - | 80 分（无重大缺陷） |
| **零缺陷交付** | +15 分 | - | QA 验收无 Bug |
| **一次性通过** | +10 分 | - | 无需返工 |
| **轻微 Bug** | - | -5 分/Bug | QA 报告轻微问题 |
| **中等问题** | - | -15 分/问题 | QA 报告需返工 |
| **严重缺陷** | - | -30 分/缺陷 | 影响核心功能 |
| **生产事故** | - | -50 分/次 | 线上故障 |
| **代码审查优秀** | +5 分/次 | - | 被作为范例 |

### 2.3 沟通分（满分 100 分）

| 评分项 | 加分规则 | 减分规则 | 评分依据 |
|--------|----------|----------|----------|
| **基准分** | - | - | 70 分（正常沟通） |
| **主动汇报** | +5 分/次 | - | 关键节点主动同步 |
| **风险预警** | +10 分/次 | - | 提前识别并上报风险 |
| **响应及时** | +5 分 | - | 消息 30 分钟内回复 |
| **协作积极** | +5 分/次 | - | 主动帮助他人解决问题 |
| **响应迟缓** | - | -5 分/次 | 消息 2h 未回复 |
| **信息不透明** | - | -10 分/次 | 隐瞒进度或问题 |
| **沟通冲突** | - | -15 分/次 | 引发团队矛盾 |

### 2.4 流程分（满分 100 分）

| 评分项 | 加分规则 | 减分规则 | 评分依据 |
|--------|----------|----------|----------|
| **基准分** | - | - | 70 分（基本遵守） |
| **文档完整** | +10 分 | - | 设计/接口/测试文档齐全 |
| **流程规范** | +5 分/次 | - | 严格遵循开发流程 |
| **会议准时** | +2 分/次 | - | 准时参加并准备充分 |
| **缺少文档** | - | -10 分/次 | 关键文档缺失 |
| **跳过流程** | - | -20 分/次 | 未经评审直接开发 |
| **未按时汇报** | - | -5 分/次 | 日报/周报缺失 |
| **门禁未通过** | - | -15 分/次 | 联调/回归未通过强行发布 |

### 2.5 创新分（满分 100 分）

## 三、统计方式

### 3.1 记录负责人

| 角色 | 职责 |
|------|------|
| **PM (胡小产)** | 负责日常评分记录、任务工时统计 |
| **QA (胡小测)** | 负责质量分数据提供（Bug 统计、验收报告） |
| **OCA (胡小豆)** | 负责最终审核、争议仲裁、月度汇总 |

### 3.2 存储位置

- **主存储：** Feishu Bitable《Agent 评分表》
- **备份存储：** `/workspace/agent-scores/` 目录下的 JSON 文件
- **公示位置：** Feishu 知识库《团队绩效看板》

### 3.3 更新频率

| 类型 | 频率 | 负责人 |
|------|------|--------|
| **日常记录** | 实时（任务完成即记录） | PM |
| **周汇总** | 每周五 18:00 | PM |
| **月结算** | 每月最后一天 | OCA |
| **季度复盘** | 每季度末 | OCA+ 老大 |

### 3.4 公平性保障

1. **数据驱动：** 所有评分必须有据可查（任务链接、Bug 报告、聊天记录）
2. **双向确认：** 评分需经被评分人确认，有异议可申诉
3. **交叉审核：** OCA 对 PM 记录进行抽查审核
4. **透明公示：** 月度评分全员可见，避免暗箱操作
5. **申诉机制：** 对评分有异议可在 3 个工作日内向 OCA 申诉

## 四、反馈机制

### 4.1 反馈方式

| 反馈类型 | 方式 | 频率 |
|----------|------|------|
| **实时反馈** | 任务完成后即时通知得分变化 | 每次任务 |
| **周反馈** | 周报中附带本周得分汇总 | 每周五 |
| **月反馈** | 1v1 面谈 + 书面报告 | 每月 |
| **季度复盘** | 正式绩效面谈 | 每季度 |

### 4.2 查看历史评分

- **个人看板：** 每个 Agent 可随时查看自己的评分历史（Feishu 知识库）
- **趋势图表：** 提供月度趋势图、维度雷达图
- **对比分析：** 可查看团队平均分作为参考（匿名）

### 4.3 自我提升支持

1. **弱项识别：** 系统自动标注得分最低的 2 个维度
2. **改进建议：** 根据弱项提供针对性建议
3. **学习资源：** 推荐相关课程、文档、案例
4. **导师配对：** 低分维度匹配高分 Agent 作为导师

## 五、激励措施

### 5.1 高分奖励

| 等级 | 标准 | 奖励 |
|------|------|------|
| **S 级** | 综合得分≥95 分 | 奖金+20%、优先晋升、额外假期 2 天 |
| **A 级** | 综合得分≥85 分 | 奖金+10%、公开表彰 |
| **B 级** | 综合得分≥75 分 | 奖金+5% |

**额外奖励：**
- **月度之星：** 当月最高分，奖励 500 元 + 奖杯
- **进步之星：** 环比提升最大，奖励 300 元
- **创新之星：** 创新分最高，奖励 400 元

### 5.2 低分后果

| 等级 | 标准 | 后果 |
|------|------|------|
| **C 级** | 综合得分 60-74 分 | 口头警告、制定改进计划 |
| **D 级** | 综合得分 50-59 分 | 书面警告、扣除奖金 10%、强制培训 |
| **E 级** | 综合得分<50 分 | 绩效不合格、调岗或淘汰 |

### 5.3 改进计划（针对 C 级及以下）

1. **问题分析：** OCA 与当事人一起分析低分原因
2. **目标设定：** 制定下月具体提升目标（至少提升 10 分）
3. **资源支持：** 提供培训、导师、工具支持
4. **跟踪检查：** 每周检查改进进度
5. **复评机制：** 连续 2 月达标可解除警告
